# Web Module

Public marketing website for Morning Alarm.

## Stack

- **Astro 5** ‚Äî static site generator (SSG), zero client JS by default
- **Tailwind CSS 3** ‚Äî utility-first styling
- **@astrojs/sitemap** ‚Äî auto-generated sitemap at build time
- **sharp** ‚Äî image optimization

Build output: `dist/` (static HTML, deployable to any CDN).

## Build & Run

```bash
cd web
npm install
npm run dev       # Dev server (http://localhost:4321)
npm run build     # Production build ‚Üí dist/
npm run preview   # Preview production build locally
```

Gradle integration: `./gradlew :web:build` delegates to `npm run build`.

## CI

- GitHub Actions runs web checks selectively when `web` files or CI workflow/scripts change.
- The current web CI contract is `npm ci`, `npm run test --if-present`, and `npm run build`.
- PRs targeting `release/*` always run the full repository suite, including web checks.

## SEO

Every page has:
- `<title>` with site name suffix
- `<meta name="description">`
- `<link rel="canonical">`
- Open Graph tags (og:title, og:description, og:image, og:url, og:type)
- Twitter Card tags (summary_large_image)

Structured data (JSON-LD):
- Home: `SoftwareApplication` schema
- FAQ: `FAQPage` schema

Static files:
- `public/robots.txt` ‚Äî allows all, points to sitemap
- Sitemap auto-generated by `@astrojs/sitemap` at `/sitemap-index.xml`

## Analytics Integration

Environment variables (build-time, PUBLIC_ prefix):
- `PUBLIC_GA_ID` ‚Äî Google Analytics measurement ID
- `PUBLIC_GSC_VERIFICATION` ‚Äî Google Search Console verification tag
- `PUBLIC_YANDEX_VERIFICATION` ‚Äî Yandex Webmaster verification tag

Set via `.env` (gitignored) or CI environment variables. Never commit secrets.

## Brand Colors

Colors come from `design-app/src/tokens/colors.ts` (MD3 color scheme). The Tailwind config maps them as `morning-*` and `night-*` color families. Any color change must be synced from the design tokens.

## Rules

- This is a **public-facing** marketing site ‚Äî no server secrets, no admin logic
- Legal pages (`/privacy`, `/terms`) must have **stable URLs** that never change
- All images must have descriptive `alt` text for SEO and accessibility
- New pages must be added to Header nav and/or Footer links as appropriate
- SEO meta (title, description, OG) is mandatory for every page
- Update this file when adding pages, changing structure, or modifying SEO setup
