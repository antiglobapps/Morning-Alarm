# Web Module

Public marketing website for Morning Alarm.

## Stack

- **Astro 5** — static site generator (SSG), zero client JS by default
- **Tailwind CSS 3** — utility-first styling
- **@astrojs/sitemap** — auto-generated sitemap at build time
- **sharp** — image optimization

Build output: `dist/` (static HTML, deployable to any CDN).

## Build & Run

```bash
cd web
npm install
npm run dev       # Dev server (http://localhost:4321)
npm run build     # Production build → dist/
npm run preview   # Preview production build locally
```

Gradle integration: `./gradlew :web:build` delegates to `npm run build`.

## Pages

| Route | File | Layout | Purpose |
|-------|------|--------|---------|
| `/` | `src/pages/index.astro` | Base | Home — hero, value props, how-it-works, download CTA |
| `/features` | `src/pages/features.astro` | Base | Morning/Night features, scenarios |
| `/screenshots` | `src/pages/screenshots.astro` | Base | App screenshot gallery |
| `/download` | `src/pages/download.astro` | Base | Store badges, system requirements, pricing |
| `/privacy` | `src/pages/privacy.md` | Legal | Privacy Policy (stable URL) |
| `/terms` | `src/pages/terms.md` | Legal | Terms of Service (stable URL) |
| `/faq` | `src/pages/faq.astro` | Base | FAQ accordion + JSON-LD FAQPage schema |

## SEO

Every page has:
- `<title>` with site name suffix
- `<meta name="description">`
- `<link rel="canonical">`
- Open Graph tags (og:title, og:description, og:image, og:url, og:type)
- Twitter Card tags (summary_large_image)

Structured data (JSON-LD):
- Home: `SoftwareApplication` schema
- FAQ: `FAQPage` schema

Static files:
- `public/robots.txt` — allows all, points to sitemap
- Sitemap auto-generated by `@astrojs/sitemap` at `/sitemap-index.xml`

## Analytics Integration

Environment variables (build-time, PUBLIC_ prefix):
- `PUBLIC_GA_ID` — Google Analytics measurement ID
- `PUBLIC_GSC_VERIFICATION` — Google Search Console verification tag
- `PUBLIC_YANDEX_VERIFICATION` — Yandex Webmaster verification tag

Set via `.env` (gitignored) or CI environment variables. Never commit secrets.

## Brand Colors

Colors come from `design-app/src/tokens/colors.ts` (MD3 color scheme). The Tailwind config maps them as `morning-*` and `night-*` color families. Any color change must be synced from the design tokens.

## File Structure

```
web/
├── public/              Static assets (robots.txt, favicon, og-image)
├── src/
│   ├── components/      Reusable Astro components (Head, Header, Footer, Hero, etc.)
│   ├── layouts/         BaseLayout (full page) + LegalLayout (markdown pages)
│   ├── pages/           File-based routing (7 pages)
│   ├── data/            Static data (site config, features, FAQ)
│   └── styles/          Global CSS (Tailwind + brand vars)
├── astro.config.mjs     Astro + integrations config
├── tailwind.config.mjs  Tailwind theme with brand colors
└── package.json         Dependencies and scripts
```

## Rules

- This is a **public-facing** marketing site — no server secrets, no admin logic
- Legal pages (`/privacy`, `/terms`) must have **stable URLs** that never change
- All images must have descriptive `alt` text for SEO and accessibility
- New pages must be added to Header nav and/or Footer links as appropriate
- SEO meta (title, description, OG) is mandatory for every page
- Update this file when adding pages, changing structure, or modifying SEO setup
