# Шаг 01. Project Structure

## Цель

Подготовить структуру репозитория для развития двух новых продуктов:
- публичного маркетингового сайта `web`
- локальной desktop-админки `desktop-admin`

## Объем работ

### 1. Добавить новые корневые модули

Нужно добавить:
- `web`
- `desktop-admin`

### 2. Подключить модули в сборку

Нужно обновить:
- `settings.gradle.kts`
- при необходимости корневой `build.gradle.kts`

### 3. Зафиксировать назначение модулей

Нужно обновить документацию:
- корневой `AGENTS.md`
- создать `web/AGENTS.md`
- создать `desktop-admin/AGENTS.md`

## Требования к модулям

### `web`
- отдельный frontend-модуль под SEO-first маркетинговый сайт
- не должен зависеть от Android/iOS-специфичных частей
- должен иметь собственный README или AGENTS с правилами структуры страниц, SEO и деплоя

### `desktop-admin`
- отдельный Kotlin/Compose Desktop модуль
- должен переиспользовать `shared` contracts
- должен быть ориентирован на внутреннее использование одним администратором

## Результат шага

После выполнения шага в проекте должны существовать:
- новый модуль `web`
- новый модуль `desktop-admin`
- актуализированная структура проекта в документации

## Критерии приемки

- проект собирается после добавления модулей
- новые модули присутствуют в `settings.gradle.kts`
- в документации проекта явно описана роль `web` и `desktop-admin`
- для каждого нового модуля создан собственный `AGENTS.md`
